# Documentation Index

Quick lookup for every `*.md` file in the repo root. Most of these are
point-in-time implementation notes ("X Complete", "X Summary") written as
work finished — they describe a snapshot, not necessarily the current state.
For current state, prefer the code, `data/`, and `git log` over an old
summary doc. The sections below flag which docs are still current.
